# Security Policy

## Reporting a vulnerability

**Do not open a public issue or PR for security problems.**

Report privately via GitHub: **Security → Advisories → "Report a vulnerability"**
(private vulnerability reporting must be enabled on this repository).

Include: affected version/commit, reproduction steps, impact assessment, and any
suggested fix. You will receive an acknowledgement within **3 business days** and a
triage decision within **7 business days**.

## Supported versions

| Version | Supported |
|---------|-----------|
| latest release | yes |
| older | no — upgrade to latest |

## Disclosure

Coordinated disclosure: we ask reporters to keep findings confidential until a fix is
released. Credit is given in the release notes unless you prefer otherwise.

## For AI agents

The operational security rulebook is [.ai/security.md](.ai/security.md) (SEC-xxx rules)
and [.ai/guardrails.md](.ai/guardrails.md). On discovering a vulnerability or leaked
secret, follow SEC-040: stop, keep it private, escalate to a human immediately.
